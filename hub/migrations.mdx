Alembic manages all schema changes via an async `env.py`. Migrations run automatically when the `api` and `worker` containers start.

## Running migrations

```bash
# Generate a new migration after modifying models
DATABASE_URL="postgresql+asyncpg://hub:hub_local@localhost:5432/hub" \
  uv run alembic revision --autogenerate -m "description"

# Apply all pending migrations
DATABASE_URL="postgresql+asyncpg://hub:hub_local@localhost:5432/hub" \
  uv run alembic upgrade head
```

<Note>
  You need the Hub's PostgreSQL container running locally before you can generate or apply migrations. Start it with `docker compose up db` if it isn't already running.
</Note>
