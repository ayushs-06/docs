## What is the Account Intelligence Hub?

The Hub is a web application for Profound's Sales and CS teams. It connects Salesforce, Gong call data (via Snowflake), and live web intelligence into a single workspace — then runs Claude AI agents to surface exactly what you need, when you need it.

<Note>
  Access requires a Profound SSO session. This documentation covers the full engineering stack: backend, data pipelines, AI agents, and frontend.
</Note>

## How data flows

Every piece of intelligence in the Hub starts with a sync from Snowflake and cascades through AI agents via SQS.

```
Salesforce / Gong (via Snowflake)
  └── sf_sync / gong_sync           ← runs every hour
        ├── company_intel            ← Claude Opus + Firecrawl
        ├── call_detect
        │     └── call_summary      ← Claude Sonnet
        │           └── action_extract ← Claude Sonnet
        ├── loss_analysis            ← Claude Opus (Closed Lost)
        └── health_score             ← no AI, Snowflake aggregates
```

On-demand flows triggered by the frontend:

```
POST /meeting-prep  →  MeetingPrepHandler  →  Claude Sonnet  →  brief
POST /zoominfo/enrich  →  ZoomInfoClient  →  contacts table
```

<Tip>
  The worker is a single Python process polling SQS. Handlers cascade by enqueuing follow-up tasks. See the [worker system](/hub/worker-system) for full retry and routing logic.
</Tip>

## Data integrations

<CardGroup cols={2}>
  <Card title="Salesforce" icon="cloud" href="/hub/snowflake-schema">
    Account records, opportunity stages, and owner metadata synced from `DBT_ANALYTICS.SALESFORCE` via Fivetran. Closed Lost detection happens at the Opportunity level.
  </Card>
  <Card title="Gong via Snowflake" icon="phone" href="/hub/snowflake-schema">
    Full call transcripts and metadata from `PROFOUND_GTM.ANALYTICS.FACT_GONG_CALLS`. No Gong API key required — all data flows through Snowflake.
  </Card>
  <Card title="HubSpot via Snowflake" icon="hubspot" href="/hub/snowflake-schema">
    Firmographic data and a three-method SF ↔ HubSpot ID mapping (confidence 1.0 → 0.9 → 0.7) stored in `id_mapping` for cross-system joins.
  </Card>
  <Card title="ZoomInfo" icon="address-book" href="/hub/external-clients">
    Contact enrichment via ZoomInfo Search (free) and Enrich (credits). Okta OAuth2 with PKCE; refresh tokens are persisted to the `settings` table.
  </Card>
  <Card title="Anthropic Claude" icon="microchip-ai" href="/hub/external-clients">
    Claude Opus for company intel and loss analysis. Claude Sonnet for call summaries, action extraction, and meeting prep. All prompts include Profound's full seller context.
  </Card>
  <Card title="Firecrawl" icon="spider-web" href="/hub/external-clients">
    Web scraping for company homepage, `/about`, `/careers`, and news search. Used exclusively by `CompanyIntelHandler` to build company profiles.
  </Card>
</CardGroup>
